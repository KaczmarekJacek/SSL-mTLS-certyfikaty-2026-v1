Na podstawie struktury z naszej poprzedniej rozmowy,
 po uruchomieniu skryptu pliki znajdują się w katalogu certs/ (lub EXPORT_DIR jeśli zmieniono):

 certs/
├── root-ca/
│   ├── rootCA.key                    ← Klucz prywatny Root CA (CHRONIĆ!)
│   └── rootCA.crt                    ← Certyfikat Root CA
│
├── intermediate-ca/
│   ├── intermediateCA.key            ← Klucz prywatny Intermediate CA
│   ├── intermediateCA.crt            ← Certyfikat Intermediate CA
│   └── certs/
│       └── ca-chain.crt              ← Łańcuch: intermediate + root (dla nginx)
│
├── server/
│   ├── server.key                    ← Klucz prywatny serwera
│   └── server.crt                    ← Certyfikat serwera
│
├── clients/
│   ├── service-a/
│   │   ├── client.key                ← Klucz prywatny klienta
│   │   ├── client.crt                ← Certyfikat klienta
│   │   └── CN.txt                    ← Common Name tego klienta
│   └── service-b/
│       └── ...
│
├── java-keystores/                   ← GOTOWE DO UŻYCIA W JAVA
│   ├── server-keystore.p12           ← Server cert + key (dla serwera)
│   ├── client-service-a-keystore.p12 ← Client cert + key (dla klienta mTLS)
│   ├── client-service-b-keystore.p12
│   └── truststore.p12                ← Root + Intermediate CA (weryfikacja)
│
└── kubernetes/
    └── tls-secrets.yaml              ← Gotowe secrety K8s
